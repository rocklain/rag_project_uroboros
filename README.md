# Ouroboros (ウロボロス) - RAG-based Architecture Visualizer

Ouroborosは、自然言語の検索クエリから関連する技術文書を検索し、その内容を元にシステム構成図やフローチャートを自動生成（可視化）するWebアプリケーションです。

## 開発の目的
「技術ドキュメントや論文を読み解き、構成図を起こす」というエンジニアの日常的な業務をAIで効率化することを目指して開発しました。単なるチャットボットではなく、「情報を検索し、視覚的なフォーマット（Mermaid.js）として出力する」という実用性にフォーカスしています。

## 主な機能
1. **RAGによる図解自動生成**
   - ユーザーのクエリをベクトル化し、Azure AI Search を用いて事前インデックス済みの技術文書から高精度にコンテキストを抽出。
   - 抽出した情報とプロンプトを Azure OpenAI (LLM) に渡し、指定された制約に従って構成図の Mermaid コードを生成させます。
2. **AI出力の自動補正エンジン (自己修復)**
   - 生成AIの出力（Mermaidコード）は揺らぎやすく、そのままでは描画エラー（構文エラー）を起こすことが多々あります。
   - 本システムでは、LLMの出力に対してPythonによる自動構文補正（自己修復処理）とリトライループを実装し、エラーに強い堅牢な図解生成を実現しています。
3. **プロンプトインジェクション対策によるセキュリティ確保**
   - LLMへのシステムプロンプトにおいて、「事前の指示を無視する」などの攻撃的・逸脱した指示を強制的に無効化する防御的プロンプト（サンドボックス化）を実装。
   - 外部からの悪意あるクエリによってシステムが想定外の振る舞い（機密情報の漏洩や別ロールへの遷移）をしないように制御しています。
4. **Azure Functions による自動データ同期バッチ**
   - Azure Functions(Timer Trigger)を用い、毎週土曜日に定期実行されるバッチ処理を構築。
   - Azure Blob Storageに新規保存された論文やドキュメントを自動でインデックス化し、AI Searchのデータソースを常に最新に保ちます。
5. **履歴の永続化と瞬時の再閲覧**
   - 生成した図解や要約はCosmosDBに保存。サイドバーからいつでも即座に過去の生成結果を呼び出せます。
6. **ゲスト用アクセス制御 (簡易認証)**
   - ポートフォリオとして安全に公開するため、共通のゲストIDとシステムキー（パスワード）による閲覧制限を設けています。IDとパスワードは面接の際に別送いたします。

## 技術スタック

### フロントエンド (UI / 描画)
- **React + TypeScript (Vite)**
- **Tailwind CSS / Lucide React**: サイバーパンク風のモダンUI構築
- **Mermaid.js**: バックエンドから受け取ったコードをブラウザ上で動的レンダリング
- **Axios**: 非同期通信処理

### バックエンド (API / RAGオーケストレーション)
- **Python 3.11 + FastAPI (uvicorn)**: 高速かつ非同期処理ベースのREST APIサーバー
- **LangChain / Azure OpenAI SDK**: プロンプト管理、LLM・Embeddingモデルの呼び出し
- **Azure SDK for Python (Search, Cosmos)**: 各種Azure PaaSとの連携
- **Pydantic**: リクエスト・レスポンスデータの厳密な型定義とバリデーション

### バッチ処理 (データパイプライン)
- **Azure Functions (Python)**: Timer Trigger による週次バッチ処理
- Blob Storage -> AI Search への定期インデクサー更新

### インフラストラクチャ (Microsoft Azure)
- **Azure App Service**: バックエンド・フロントエンドのホスティング（VNet統合）
- **Azure AI Search**: 独自データのベクトル検索基盤
- **Azure OpenAI Service**: Embedding（ベクトル化）および Chat（コード生成）モデル
- **Azure Cosmos DB (NoSQL)**: 履歴データの高速保存。Private Endpointを用いてインターネットから隠蔽し、セキュアな閉域網通信を構成。

## 技術的な工夫点（アピールポイント）
* **AI出力の不安定さをコードでカバーした点**
  * LLMに「Mermaid構文で出力して」と指示するだけでは、存在しないノードへの矢印など仕様外のコードを出力しがちです。これに対し、「プロンプトによる制約」「Pythonでの正規表現を用いた自動補正」「失敗時のAI自己リトライ」という3段構えのサニタイズロジックを自作しました。
* **クラウドネイティブかつセキュアなインフラ設計**
  * Cosmos DB をパブリックネットワークから遮断し、Private Endpoint と App Service の VNet 統合を用いて、セキュアな閉域網通信でシステムを構築しています。
